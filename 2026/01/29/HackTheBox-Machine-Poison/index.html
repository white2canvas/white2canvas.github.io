HackTheBox Machine Poison

John Doe
2026-01-29
base64, freebsd, lfi, ssh, vnc

1. 信息搜集

1.1 端口扫描

对目标机器进行TCP半连接的端口和服务扫描：只有22和80端口开放。

  • -sC : 使用默认的NSE脚本进行漏洞检测、服务识别等。
  • -sV : 对开放的端口进行软件及版本探测。
  • --min-rate=10000 : 每秒至少发送10000个数据包。
  • -p- : 全端口扫描。
  • -v : 显示扫描过程中的详细信息，可以使用vv或者vvv获取更详细输出。
  • -oN ./nmap/poison.nmap：以文本格式保存扫描结果。

1.2 Web枚举

访问Web站点，首页是一个测试页面，可以输入PHP文件名。

根据首页显示的各个文件名依次输入测试，在listfiles.php中发现备份凭证文件名。

同时这儿存在本地文件包含漏洞：在/etc/passwd文件中发现用户charix。

继续包含上面泄露的pwdbackup.txt凭证文件：发现是base64格式的密文。

2. 初始立足

2.1 密码破解

上面pwdbackup.txt中说明至少解密了13次，尝试用cyberchef解密，需要多次解密。

写循环base64解密脚本进行解密，直到解密失败，得到密码Charix!2#4%6&8(0。

2.2 SSH连接

从/etc/passwd中得到用户名charix，从pwdbackup.txt解密得到密码Charix!2#4%6&8(0。

尝试用SSH进行连接，用户名密码如上，连接成功。

3. 权限提升

3.1 凭证传输

在用户家目录下，除了user.txt文件，还看到一个secret.zip压缩包。

利用scp将secret.zip压缩包传输到本地。

解压发现存在密码，有两种破解方式，密码复用或者字典爆破。

尝试用charix用户的密码进行解压，解压成功。

3.2 VNC连接

查询TCP活动端口信息：5801（浏览器访问）和5901（客户端连接）是VNC服务相关端口。

查看相关进程启动的完整命令（ww参数）。

详细解释上面XVNC的部分启动命令。

  • :1 : 指定显示编号为1，对应5901端口。
  • -desktop X : 设置桌面名称为X。
  • -httpd /usr/local/share/tightvnc/classes : 指定Java VNC查看器的HTTP服务目录。
  • -auth /root/.Xauthority : 使用指定文件进行X授权认证。
  • -rfbauth /root/.vnc/passwd : 指定VNC密码文件路径，此密码文件需要用vncpasswd生成。
  • rfbport 5901 : 指定RFB协议监听端口为5901。
  • -localhost : 只允许本地连接。

现在确定目标机器上存在VNC服务，并且只有本地可以连接。

利用SSH做动态端口转发，设置代理端口9999，通过SSH隧道将流量转发到目标机器。

  • -C : 压缩数据传输，提高传输速度。
  • -N : 不执行远程名，只建立连接但不登录。
  • -f : 建立连接后在后台运行。

现在通过本地的9999端口作为socks5代理进行VNC连接。

4. 总结提升

4.1 LFI RCE

上面直接通过SSH建立了初始连接，但是存在本地文件包含漏洞，包含HTTP日志也可以成功RCE。

包含Apache配置文件/usr/local/etc/apache24/httpd.conf来寻找访问日志路径。

继续包含该访问日志，包含成功。

访问Web站点任意路由，修改User-Agent为PHP木马。

包含日志进行命令执行，命令执行成功。

4.2 SSH代理

搭建环境并测试三种代理方式：本地端口转发、远程端口转发和动态端口转发。

攻击机：192.168.112.135（22/SSH）

边界服务器：192.168.112.152/10.211.55.138（22/SSH 8000/HTTP/本地监听）

内网服务器：10.211.55.137（8000/HTTP/任意地址）

本地端口转发：将本地指定端口的流量通过SSH隧道转发到远程机器的指定端口。

通过本地的8888端口访问边界服务器的Web服务，通过本地的9999端口访问内网服务器的Web服务。

访问测试，通过本地的8888端口访问边界服务器的Web服务成功。

通过本地的9999端口访问内网服务器的Web服务成功。

远程端口转发：将远程指定端口的流量通过SSH隧道转发到本地（或其他地址）指定端口。

访问测试，通过本地的8888端口访问边界服务器的Web服务成功。

通过本地的9999端口访问内网服务器的Web服务成功。

动态端口转发：本地开放一个代理端口，代理端口通过SSH隧道将流量转发到目标机器。

访问测试，通过socks5代理端口访问边界服务器的Web服务。

通过socks5代理端口访问内网服务器的Web服务。
