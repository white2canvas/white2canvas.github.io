HackTheBox Machine Pterodactyl

John Doe
2026-02-16

1. 信息搜集

1.1 端口扫描

对目标机器进行端口和服务扫描：仅开放22和80端口，只能从Web站点入手。

添加本地DNS解析到/etc/hosts文件，访问Web站点测试。

1.2 Web枚举

在Web主页，发现了一个play.pterodactyl.htb子域名，添加到本地/etc/hosts后访问重定向到主域名。

在前台的Web站点有个Changelog按钮，点进去发现此站点安装了Pterodactyl Panel面板。

已知Pterodactyl Panel版本为1.11.10，在Github Advisory发现该版本存在未授权RCE漏洞。

现在的问题在于需要找到Pterodactyl Panel路径或者子域名。

1.3 域名爆破

用常规字典对目标站点进行目录扫描，未发现可能是Pterodactyl Panel地址的路径。

在进行子域名爆破之后，发现了panel.pterodactyl.htb子域名。

添加到/etc/hosts本地解析之后访问发现，确定是Pterodactyl Panel站点。

2. 初始立足

2.1 CVE-2025-49132

在Exploit DB找到了关于该漏洞的EXP。

上面的EXP核心就是一个Get传参，尝试手工利用：MySQL用户密码泄露pterodactyl:PteraPanel。

找到了关于本地包含导致RCE的EXP，可以直接进行命令执行。

接着利用/bin/bash反弹Shell拿到伪Shell。

拿到Shell之后查看当前系统信息，发现是OpenSUSE系统。

3. 横向移动

3.1 MySQL凭证泄露

用上面的包含得到凭证进行连接，查询数据。

3.2 Bcrypt哈希爆破

将得到的两个密码哈希保存到本地，用john进行爆破。

爆破出来一个密码明文为!QAZ2wsx。

尝试用/etc/passwd中的headmonitor和phileasfogg3用户结合上面的密码进行SSH连接。

4. 权限提升

4.1 CVE-2025-6018

在目标机器上存在Pam 1.3版本。

在Exploit DB中找到了关于PAM本地提权的漏洞EXP。

用上面的EXP脚本进行利用，提权失败。

在Github找到了另一个分步利用的EXP。

阶段一：在攻击机中创建XFS镜像，并且传输到目标机器。

阶段二：在目标机器上进行PAM注入，登出之后重新认证。

阶段三：UDisk2漏洞利用。
